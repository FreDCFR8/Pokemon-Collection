# UI Guidelines

## Status

Phase 0 draft.

## Principle

The UI is new and may not copy the previous application layout or styling.

## Primary target

Design first for iPhone Safari.

## Navigation

Top bar:

- hamburger menu
- search
- AI Assistant entry point, inactive until approved

Primary sections:

- Dashboard
- Collection
- Sets
- Wishlist
- Pokédex

Binder is deferred.

## Component principles

- small reusable components
- predictable spacing
- readable tap targets
- no hidden business logic in visual components
- accessible contrast
- clear loading and empty states
- explicit error states

## Mobile rules

- avoid dense desktop tables on mobile
- avoid rendering huge lists at once
- avoid unexpected layout shifts
- prioritize one clear action per screen
