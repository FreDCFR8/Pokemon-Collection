# Phase 4H canonical sets CSV content

Phase 4H voegt de eerste canonical sets CSV-content toe voor review. Deze fase levert alleen een handmatig samengestelde CSV v1 op en voert geen Supabase import uit.

De CSV v1 mag onvolledig zijn. Lege velden betekenen dat de waarde nog niet betrouwbaar is bevestigd; er zijn daarom geen fictieve release dates, totals of URL's toegevoegd.

De import volgt pas later na review van de CSV-content. Tot die tijd blijft `cards_catalog.set_name` alleen een validatiehulp en geen canonical source voor setgegevens.
